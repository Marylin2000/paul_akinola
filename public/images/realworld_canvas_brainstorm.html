The big idea — CareerWorld Sandbox
From gamified learning → real career simulation.
After completing the 5-mission story path, kids unlock a real-world sandbox canvas for each career. It's not pretend anymore — they write actual code that runs, diagnose simulated patients with real medical logic, manage real farm sensors, design with real tools. The game becomes a career launchpad.
What changes at each stage
1
Story missions (ages 4–10)
Cartoon narrative, guided missions, drag-and-drop activities. Introduces career concepts through play.
2
Challenge missions (ages 10–12)
Timed quizzes, decision trees, real data. Kids see consequences of their choices. Still guided but harder.
3
🆕 Real-world sandbox (ages 12–14+)
Live coding environment, patient simulator, farm sensor dashboard, design canvas. No guardrails — real tools, real output, real stakes.
4
🆕 Portfolio & certification (any age)
Sandbox output saved to portfolio. Real certificates. Shareable proof of skill that means something to schools and future employers.

Sandbox levels for developer
🟢
Level 1 — Block IDE (visual)
Ages 8–11
Scratch-style drag-and-drop coding blocks. Kids assemble logic visually. Output is real — a working mini app runs in a WebView. Introduces functions, loops, conditionals without syntax.
🟡
Level 2 — Guided text IDE
Ages 11–12
Real JavaScript/Python but with inline hints and autocomplete. Kids write actual code. Output runs in a sandboxed Node.js environment. Tasks: build an API, connect to a JSON database, handle errors.
🔴
Level 3 — Full sandbox (production sim)
Ages 12–14+
Deploy a real app. Manage a live SQLite database. Read server logs. Handle simulated traffic spikes. Introduce CI/CD concepts. Kids get a real URL they can share. This is junior developer territory.

Can React Native do this? — Yes. Here's how.
✅ What RN handles natively
WebView (runs full web IDE, Jupyter notebook, any HTML tool)
react-native-svg (designer canvas shapes & paths)
Animated API (vitals monitor, farm sensor animations)
AsyncStorage / SQLite (local data persistence)
PanResponder / Gesture Handler (drag-and-drop)
Camera / AR (future: scan crops, scan patients)
⚡ What needs a backend service
Code execution (sandboxed Node.js / Python — use Piston API or Judge0)
Real deployment (use a managed sandbox like Render.com or Railway free tier)
AI patient responses (Claude API — lightweight call per question)
Database sandbox (Supabase or PocketBase — free tier per user)
Real-time farm sensors (simulated via WebSocket server)
Architecture for each canvas
💻 Developer canvas
RN WebView → Monaco Editor (VS Code's editor, runs in WebView) → Code sent to Piston API (open source, free, runs Node/Python/Go in sandboxed containers) → Output streamed back → Deploy button calls Railway.app API → Kid gets a real URL like careerworld.app/amara-123
🩺 Doctor canvas
Patient data is a JSON scenario file (age, symptoms, vitals, correct diagnosis). Vitals are animated with Reanimated 3. Diagnosis logic is a decision tree stored locally — no API needed for basic scenarios. Claude API powers the "ask the patient a question" feature — lightweight, 1 call per question, cached responses for common questions.
🌾 Farmer canvas
IoT sensors are simulated — a background timer updates soil/weather values with realistic variation algorithms. Weather events are seeded from a scenario JSON. Market prices use a simple supply/demand model running locally. No external API needed — works fully offline.
🎨 Designer canvas
react-native-svg for drawing. react-native-view-shot to export canvas as PNG. Colour picker via react-native-color-picker. Font loading via expo-font. AI client feedback via Claude API — "here is my design [image], give feedback as a Nigerian brand client." Export saves to device + CareerWorld portfolio.
