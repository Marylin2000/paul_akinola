The big idea — CareerWorld Sandbox
From gamified learning → real career simulation.
After completing the 5-mission story path, kids unlock a real-world sandbox canvas for each career. It's not pretend anymore — they write actual code that runs, diagnose simulated patients with real medical logic, manage real farm sensors, design with real tools. The game becomes a career launchpad.
Learning progression
1
Story missions (ages 4–10) — Cartoon narrative, guided drag-and-drop. Introduces career concepts through play.
2
Challenge missions (ages 10–12) — Timed quizzes, decision trees, real data. Still guided but harder.
3
🆕 Real-world sandbox (ages 12–14+) — Live coding, patient simulator, farm sensor dashboard, design canvas. Real tools, real output, real stakes.
4
🆕 Portfolio & certification — Sandbox output saved to portfolio. Real certificates. Shareable proof of skill.

3 sandbox levels
🟢
Level 1 — Block IDE (visual)
Ages 8–11
Scratch-style drag-and-drop blocks. Output is real — a working mini app runs in a WebView. Introduces functions, loops, conditionals without syntax anxiety.
🟡
Level 2 — Guided text IDE
Ages 11–12
Real JavaScript/Python with inline hints and autocomplete. Kids write actual code. Runs in sandboxed Node.js. Tasks: build an API, connect to a JSON database, handle errors.
🔴
Level 3 — Full production sandbox
Ages 12–14+
Deploy a real app. Manage a live SQLite database. Read server logs. Handle simulated traffic spikes. Kids get a real URL they can share with family. This is junior developer territory.

Can React Native do this? Yes. Here's how.
✅ RN handles natively
WebView — runs full Monaco IDE, any HTML tool
react-native-svg — designer canvas shapes
Reanimated 3 — vitals monitor animations
AsyncStorage / SQLite — local data
Gesture Handler — drag-and-drop
Camera / AR — scan crops, scan patients (future)
⚡ Needs backend service
Code execution → Piston API (open source)
Real deployment → Railway.app free tier
AI patient/client → Claude API
Database sandbox → Supabase free tier
Farm sensors → simulated WebSocket server
Architecture per canvas
💻 Developer
RN WebView → Monaco Editor (VS Code's engine, runs in WebView) → code sent to Piston API (sandboxed Node/Python containers) → output streamed back → Deploy button calls Railway.app API → kid gets real URL: careerworld.app/amara-123
🩺 Doctor
Patient data = JSON scenario file. Vitals animated with Reanimated 3. Diagnosis = local decision tree JSON. Claude API powers "ask the patient" — ~$0.003/question. Common questions are cached so most sessions need zero API calls.
🌾 Farmer
IoT sensors = background timer updating values with realistic variance algorithms. Weather events seeded from scenario JSON. Market prices use simple supply/demand model. 100% offline — no API needed.
🎨 Designer
react-native-svg for drawing. react-native-view-shot exports canvas as PNG. AI client feedback via Claude API — pass the image, get brand critique. Export saved to device + CareerWorld portfolio.
